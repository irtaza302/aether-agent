# Changelog

All notable changes to the Aizen AI CLI will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- **Plugin System**: Developers can now add custom tools to Aizen by placing python scripts in `~/.aizen/plugins/`.
- **MCP Support**: Integration with the Model Context Protocol (MCP) to load external servers and tools.
- **Auto-Compaction**: Automatic context pruning when the conversation exceeds 80% of the token limit to prevent massive API bills and token exhaustion.
- **Web Search Fallback**: `web_search` tool now gracefully falls back to a DuckDuckGo HTML parser if standard integrations fail.

### Changed
- **Architectural Overhaul**: Split the massive monolithic codebase into a highly modular package (`aizen/tools/`, `aizen/agent.py`, `aizen/context.py`).
- **Concurrent Execution Safety**: Implemented thread locks across all tool file writes to ensure data integrity when tools are executed concurrently.
- **CLI Streaming**: Improved stream rendering and internal tool dispatch logic.

### Fixed
- Re-architected JSON parsing for tool dispatch to natively heal broken trailing commas or invalid whitespace generated by certain LLMs.
- Addressed multiple edge cases in context size tracking and estimation where tool usages were not being perfectly counted.

## [2.3.0] - 2024-05-15
### Added
- Initial support for dynamic model loading.
- Git-dirty file write warnings to prevent accidental overwrites.
